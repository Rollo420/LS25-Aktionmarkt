0.1. About Laravel

Laravel is a web application framework with expressive, elegant syntax. We believe development must be an enjoyable and creative experience to be truly fulfilling. Laravel takes the pain out of development by easing common tasks used in many web projects, such as:

  • Simple, fast routing engine.
  • Powerful dependency injection container.
  • Multiple back-ends for session and cache storage.
  • Expressive, intuitive database ORM.
  • Database agnostic schema migrations.
  • Robust background job processing.
  • Real-time event broadcasting.

Laravel is accessible, powerful, and provides tools required for large, robust applications.

1. Laravel 11 Sail mit Bootstrap, TailwindCSS und Vite einrichten

1.1. 1️⃣ Laravel 11 mit Sail aufsetzen

1.1.1. Laravel-Projekt erstellen

Falls noch kein Laravel-Projekt existiert, erstelle eines mit:

curl -s "https://laravel.build/meinprojekt" | bash
cd meinprojekt

Falls du bereits ein Laravel-Projekt hast, stelle sicher, dass Sail installiert ist:

composer require laravel/sail --dev
php artisan sail:install

Starte Sail mit:

sail up -d

1.2. 2️⃣ Bootstrap SCSS in Laravel 11 mit Sail installieren

1.2.1. Bootstrap & Abhängigkeiten installieren

sail npm install bootstrap @popperjs/core sass

1.2.2. SCSS-Datei für Bootstrap erstellen

touch resources/css/app.scss

Öffne resources/css/app.scss und füge hinzu:

@import "bootstrap/scss/bootstrap";

Falls du Bootstrap-Variablen überschreiben möchtest, tue das vor dem Import:

$primary: #3490dc;
$secondary: #ffed4a;
@import "bootstrap/scss/bootstrap";

1.2.3. Vite für SCSS konfigurieren

Öffne die Datei vite.config.js und stelle sicher, dass SCSS und JS eingebunden sind:

import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/css/app.scss', 'resources/js/app.js'],
            refresh: true,
        }),
    ],
});

1.2.4. Bootstrap JavaScript importieren

Öffne resources/js/app.js und füge hinzu:

import 'bootstrap';
import * as Popper from "@popperjs/core";
window.Popper = Popper;

1.2.5. Vite in Blade-Template einfügen

Öffne dein Hauptlayout resources/views/layouts/app.blade.php und füge im <head>-Bereich hinzu: s

@vite(['resources/css/app.scss', 'resources/js/app.js'])

1.2.6. Vite starten

Falls dein Laravel Sail-Container schon läuft, starte Vite mit:

sail npm run dev

Falls der Container noch nicht läuft, starte ihn zuerst:

sail up -d
sail npm run dev

🔄 Jetzt werden Änderungen an SCSS automatisch erkannt und die Seite wird neu geladen! 🚀

1.3. 3️⃣ LiveReload mit Vite einrichten

Damit Vite automatische Live-Reloads durchführt:

  1. Stelle sicher, dass vite.config.js das refresh: true enthält (siehe oben).
  2. Starte den Vite-Entwicklungsserver:
sail npm run dev
  1. Prüfe, ob im Browser-Netzwerk-Tab (F12 → Network) die Vite-WS-Verbindung (ws://) besteht.

Falls du nicht in der Standard-Umgebung von Sail arbeitest, könnte ein VITE_HMR_HOST notwendig sein. Setze es in .env:

VITE_HMR_HOST=localhost

Danach Sail neu starten:

sail down
sail up -d
sail npm run dev

1.4. 4️⃣ TailwindCSS mit Laravel 11 Sail installieren

1.4.1. TailwindCSS & Abhängigkeiten installieren

sail npm install -D tailwindcss postcss autoprefixer
sail npx tailwindcss init -p

1.4.2. Tailwind konfigurieren

Öffne tailwind.config.js und passe den content-Bereich an:

/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./resources/**/*.blade.php",
    "./resources/**/*.js",
    "./resources/**/*.vue",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};

1.4.3. Tailwind in CSS einbinden

Bearbeite resources/css/app.scss oder resources/css/app.css und füge ein:

@tailwind base;
@tailwind components;
@tailwind utilities;

1.4.4. Vite in Blade-Template einfügen

Falls noch nicht geschehen, öffne resources/views/layouts/app.blade.php und füge hinzu:

@vite(['resources/css/app.scss', 'resources/js/app.js'])

1.4.5. Vite starten

sail npm run dev

🔥 Jetzt ist Laravel 11 mit Sail, Bootstrap SCSS, TailwindCSS und LiveReload korrekt eingerichtet! 🚀

2. Fehlerbehebungen in Laravel 11 mit Sail

2.1. Fehler: sh: 1: vite: not found

2.1.1. Problem

Wenn du versuchst, Vite mit sail npm run dev zu starten, aber die Fehlermeldung sh: 1: vite: not found erhältst, bedeutet das, dass Vite nicht installiert ist.

2.1.2. Lösung

Führe folgende Befehle aus:

sail npm install
sail npm install vite laravel-vite-plugin --save-dev

Danach starte Vite erneut:

sail npm run dev

3. Fehlerbehebungen in Laravel 11 mit Sail

3.1. Fehler: sh: 1: vite: not found

3.1.1. Problem

3.1.2. Lösung

sail npm install vite laravel-vite-plugin --save-dev

Danach starte Vite erneut:

3.5. Fehler: Automatisches Reloading funktioniert nicht in Laravel 11 mit Vite

3.5.1. Problem

Nach Änderungen an SCSS oder JS lädt die Seite nicht automatisch neu.

3.5.2. Lösung

Stelle sicher, dass Vite richtig konfiguriert ist:

  1. vite.config.js anpassen
    import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/css/app.scss', 'resources/js/app.js'],
            refresh: true,
        }),
    ],
});
  2. Vite in app.blade.php einfügen
    @vite(['resources/css/app.scss', 'resources/js/app.js'])
  3. Vite erneut starten
    sail npm run dev

Falls es nicht funktioniert, lösche den node_modules-Ordner und installiere alles neu:

sail npm install
sail npm run dev

3.6. Automatisches Starten von sail npm run dev beim Docker-Start

3.6.1. Problem

Nach jedem Neustart von Docker oder Sail muss man sail npm run dev manuell starten.

3.6.2. Lösung

Du kannst das automatisch beim Start von Sail mit einem eigenen entrypoint.sh-Skript lösen.

  1. Erstelle ein Skript im Projektverzeichnis:
    touch sail-entrypoint.sh
chmod +x sail-entrypoint.sh
  2. Öffne sail-entrypoint.sh und füge folgendes hinzu:
    #!/bin/sh
sail up -d && sail npm run dev
  3. Füge die Datei in deine docker-compose.override.yml ein, um sie beim Start auszuführen:
    version: '3'
services:
    laravel.test:
        entrypoint: ["/bin/sh", "-c", "./sail-entrypoint.sh"]
  4. Starte Sail neu:
    sail down
sail up -d

🔄 Jetzt startet sail npm run dev automatisch mit Docker! 🚀
